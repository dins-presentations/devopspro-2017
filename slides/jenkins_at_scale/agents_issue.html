Challenge: So it is release time, build queue is getting longer

Challenge: Each stage requires a separate environment with different requirements

Solution: Using the cloud infrastructure (e.g. Apache Mesos)

### Низкая утилизация ресурсов Проблема: Во время релиза очередь на Jenkins постоянно занята и требует много ресурсов, а в остальное время эти ресурсы простаивают. Проблема: Каждый этап непрерывной интеграции требует свое собственное окружение разного размера. И оно также простаивает. Решение: Используем облачную инфраструктуру (например Mesos) для динамического создания агентов и окружений Это позваляет использовать более ресурсы эффективно.
